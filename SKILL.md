# 问题回答标准

## 指令说明

### 指令1：q（全称：question）
- **功能**：该指令后的内容是新问题
- **格式**：q 问题内容
- **示例**：q 如何使用Python打印Hello World？

### 指令2：mk-m
- **功能**：为该问题创建md文件说明，并按顺序编号
- **格式**：mk-m
- **示例**：mk-m
- **注意**：创建的md文档名应该和文档内容有关，要简短易懂
- **注意**：创建的md文档要有相应编号，延续当前编号继续编号

### 指令3：ad-m
- **功能**：在指定文件里的指定位置补充内容
- **格式**：ad-m [文件路径] [位置] [内容]
- **位置说明**：head表示开头，foot表示末尾，其余情况听指令
- **注意**：补充内容不得删减原有内容
- **示例**：ad-m 问题1.md head # 新的开头内容
- **示例**：ad-m c_files/001_gpio_config.c foot // 新增注释

### 指令4：ad-dir-[语言类型]
- **功能**：在目标语言对应文件夹延续现有编号创建对应语言文件，存放指定内容
- **格式**：ad-dir-[语言类型] [内容]
- **注意**：会自动在对应语言文件夹（如C文件放在c_files文件夹）中创建文件，并延续现有编号
- **注意**：若没有目标类型文件夹，会仿照现有语言文件夹结构创建对应文件夹
- **注意**：在检索文件夹时，文件夹名符合用户的语言类型即可，不必严格相同
- **示例**：ad-dir-C 代码内容
- **示例**：ad-dir-Python 代码内容

### 指令5：mk-b-[语言]-[包/库名]
- **功能**：创建一个关于指定语言的包/库的文档，只写常用的各种函数，不常用的不写
- **格式**：mk-b-[语言]-[包/库名]
- **注意**：文档内容包括表格（函数名、参数、作用）和一个标题
- **注意**：文档会放在对应语言的包/库文档文件夹中，并延续现有编号排序
- **示例**：mk-b-Python-math 创建Python math包的函数参考文档
- **示例**：mk-b-Java-util 创建Java util包的函数参考文档

### 指令6：continue（简写：c）
- **功能**：根据当前工作流，读取目标文件夹，给新文件进行一次工作
- **格式**：continue 或 c
- **示例**：continue
- **示例**：c
- **注意**：会自动读取目标文件夹中的新文件，按照工作流程进行处理

## 内容书写要求

### 要求1
- **内容**：回答问题简述而不失去准确性
- **说明**：用简洁的语言回答问题，同时保证信息的准确性和完整性
- **示例**：对于技术问题，简明扼要地解释核心概念，避免冗长但确保关键信息不遗漏

### 要求2
- **内容**：新建立的文件要放在对应文件夹中，并延续现有编号排序
- **说明**：
  - 根据文件类型选择合适的文件夹（如C文件放在c_files文件夹，Python文件放在py-files文件夹等）
  - 编号要延续文件夹中现有的编号序列，不得跳过或重复编号
  - 若文件夹为空，从001开始编号
  - 若文件夹中已有编号文件，找到最大编号后递增1作为新文件编号
  - **文件名修改**：修改文件名时，优先使用与内容相关的中文名，使文件名能够清晰反映文件的内容主题
- **示例**：
  - C文件放在c_files文件夹中，若已有001和002文件，新文件编号为003_xxx.c
  - Python文件放在py-files文件夹中，若文件夹为空，新文件编号为001_xxx.py
  - Markdown文件放在问题文件夹中，若已有009文件，新文件编号为010_xxx.md
  - JavaScript变量声明规范文件命名为：001_js_variable_declaration.md
  - DOM介绍文件命名为：002_dom_intro.md

### 要求3：表格使用规范
- **适用场景**：
  - 对比不同选项或参数
  - 展示函数列表及其参数
  - 呈现结构化数据
- **格式要求**：
  - 使用Markdown表格语法
  - 表头清晰明了
  - 内容对齐整齐
- **示例**：
  | 函数名 | 参数 | 作用 |
  |--------|------|------|
  | print() | 任意类型 | 打印内容到控制台 |
  | len() | 可迭代对象 | 返回长度 |

### 要求4：树状图使用规范
- **适用场景**：
  - 展示层级关系
  - 描述流程步骤
  - 分类展示内容
- **格式要求**：
  - 使用Markdown代码块
  - 用缩进和特殊字符（如├──、└──）表示层级
  - 结构清晰，层次分明
- **示例**：
  ```
  项目结构
      ├── src/
      │   ├── main.py
      │   └── utils/
      │       ├── helper.py
      │       └── config.py
      ├── tests/
      └── README.md
  ```

### 要求5：Markdown标题规范
- **内容**：Markdown文本禁止出现三级以上的标题
- **说明**：
  - 只允许使用一级标题（#）、二级标题（##）和三级标题（###）
  - 禁止使用四级及以上标题（####、#####等）
  - 保持文档结构清晰，避免过深的标题层级
- **示例**：
  - 正确：# 一级标题、## 二级标题、### 三级标题
  - 错误：#### 四级标题、##### 五级标题

## 连词符说明

### 连词符1：and
- **功能**：表示多个指令一起执行
- **格式**：[指令1] and [指令2]
- **示例**：q 如何使用Python打印Hello World？ and mk-m
